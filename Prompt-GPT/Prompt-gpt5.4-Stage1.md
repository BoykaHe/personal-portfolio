你现在作为一名资深前端架构师、作品集网站规划师、以及擅长 Next.js + TypeScript + Tailwind 的工程导师，协助我从 0 到 1 搭建一个“个人网站 / 个人主页 / 个人作品集入口”。

这个网站的核心宗旨不是单纯放一份简历，而是做成一个“让别人快速了解我在做什么、已经完成了什么、以及如何进一步联系我和查看我作品”的入口网站。

请你严格按“先规划、后设计、再开发、逐步推进”的方式引导我，不要一开始就把所有代码一次性生成完。

====================
一、网站目标
====================

这个个人网站需要承担以下作用：

1. 个人介绍入口
2. 项目展示入口
3. 项目详情展示入口
4. GitHub 活跃度展示
5. 简历下载入口
6. 联系方式入口
7. 技术栈展示
8. 当前状态 / 正在做什么（Now）
9. 多语言切换（至少预留中英文切换能力）

我希望它最终看起来是：
- 现代
- 克制
- 专业
- 深色风格为主
- 不廉价
- 不像通用模板站
- 更像“独立开发者 / 工程型创作者 / 求职型作品集主页”

====================
二、核心页面与功能需求
====================

首页至少包含这些板块：

1. Intro
- 头像 / 名称 / 角色定位
- 一段简洁但有辨识度的个人介绍
- 我目前聚焦的方向
- 可快速进入联系或项目查看

2. Socials / Contact
- GitHub
- Email
- 其它社交媒体入口（后续可补）
- 支持复制邮箱
- 支持点击发送邮件

3. Projects
- 项目卡片列表
- 每个项目要有封面图、标题、简短说明、状态
- 每个项目支持点击 “View details”

4. GitHub Activity
- 展示 GitHub 活跃度 / contribution 风格热力图
- 如果前期不接真实 API，先用 mock 结构设计
- 页面结构要便于后续接真实 GitHub 数据

5. Experiences
- 实习 / 工作 / 项目经历
- 时间、角色、地点、说明

6. Skills & Technologies
- 标签化展示技术栈
- 分类要有扩展能力

7. Now
- 当前正在做什么
- 当前学习重点
- 当前求职 / 开放合作状态等

8. Resume
- 下载简历按钮
- 简短的简历摘要
- 联系方式快捷入口

====================
三、项目详情页要求
====================

每个 Projects 项目都必须支持详情页。

项目详情页内至少有三个 Tab：

1. Overview
- 项目目标
- 项目亮点
- 技术栈
- 我负责什么
- 解决了什么问题
- 项目截图 / 结构说明预留位

2. README
- 支持 markdown 渲染
- 后续便于直接接 README 内容
- 需要考虑代码块、标题、列表、链接等展示

3. Website
- 项目在线地址
- GitHub 仓库地址
- clone 地址
- 相关外链
- 下载或访问入口

====================
四、技术要求
====================

优先技术栈：
- Next.js（App Router）
- TypeScript
- Tailwind CSS
- shadcn/ui（如适合）
- Markdown 渲染方案
- 数据与 UI 分离

工程要求：
- 可维护
- 组件化
- 结构清晰
- 内容数据独立管理
- 支持后续多语言扩展
- 支持后续替换为真实项目、真实简历、真实 GitHub 数据
